```sql
SELECT

    concat(ROUND(SUM(index_length) / 1024 / 1024, 2) + ROUND(SUM(data_length)  / 1024 / 1024, 2), ' MB') "total_size",
```

### Задание 3*

Самостоятельно изучите, какие типы индексов используются в PostgreSQL. Перечислите те индексы, которые используются в PostgreSQL, а в MySQL — нет.

*Приведите ответ в свободной форме.*
```
Типы индексов в PostgreSQL:
B-Tree — стандартный индекс «по умолчанию» (есть и в MySQL). 
    Подходит для данных, которые можно отсортировать.

Hash — хранит хэш-код значения. 

GiST (Generalized Search Tree) — Используется для полнотекстового поиска и геоданных.

SP-GiST (Space-Partitioned GiST):
    Для данных, которые можно разбивать на неперекрывающиеся области (например, деревья или префиксы).

GIN (Generalized Inverted Index) — инвертированный индекс. Для массивов и JSONB.

BRIN (Block Range Indexes):
    Хранит только мин/макс значения для блоков данных. 
    Идеален для гигантских таблиц (терабайты), где данные упорядочены по времени.

MySQL (на движке InnoDB) практически ограничен B-Tree и Hash (только для адаптивных нужд) и частичной поддержкой R-Tree (Spatial).
Вот чего в MySQL нет:

GIN (Generalized Inverted Index):
    Самое большое отличие. 
    В MySQL нет эффективного способа индексировать содержимое JSON «внутри» так,
    как это делает GIN, позволяя искать по любому ключу или значению в документе.

BRIN:
    В MySQL нет аналога, который позволял бы индексировать миллиарды строк,
    занимая всего несколько мегабайт памяти за счет хранения диапазонов блоков.

GiST / SP-GiST:
    Postgres использует их для индексации пересекающихся геометрий,
    сетевых адресов (inet) и интервалов времени.
```
